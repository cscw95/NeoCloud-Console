# NeoCloud Biz 콘솔 — 영업 메뉴 사용법 매뉴얼

대상 화면: **파이프라인 · CRM** / **견적 · RFQ** / **계약 관리** (사이드바 "영업" 그룹)
접속: `http://127.0.0.1:8090/biz/` · 상단 배지 **"Control-Plane 연동"**(초록)이면 라이브 모드,
**"Control-Plane 미연동"**(주황)이면 mock 폴백 모드로 동작한다.

> 라이브 모드에서 계약 전환은 **실제 Control-Plane 테넌트·랙 주문을 생성**한다.
> 전환 주문은 운영 콘솔의 승인 게이트(7단계)를 통과해야 개통(delivered)된다.

## 1. 파이프라인 · CRM (`#/pipeline`)

### 화면 구성
| 영역 | 내용 |
|------|------|
| KPI 밴드 | 가중 파이프라인($M)·분기 승률·평균 딜 사이즈·평균 사이클 |
| 스테이지 보드 | 5레인: **리드 → 상담 → 제안/견적 → 협상 → 계약 전환**. 레인 헤더에 건수·가중 ARR 자동 집계 |
| 딜 카드 | 고객사·랙 수·확률·**진행률 바**(세부 단계 완료율)·다음 액션·담당자 |
| 딜 상세 패널 | delta-corp 등 라이브 딜의 규모/배치/활동 로그 (Control-Plane 연동) |

### 사용법
1. **리드 등록** — `+ 리드 등록` → 고객명(필수)·랙 수·확률 입력 → 보드 "리드" 레인에 즉시 표시.
2. **딜 상세 열기** — 보드에서 **고객사 카드를 클릭**하면 딜 상세 모달이 열린다.
3. **스테이지 변경** — 모달 상단 **스테이지 스텝퍼**에서 원하는 단계를 클릭하면 앞/뒤 어느 방향이든
   즉시 이동하고, 보드 컬럼도 실시간으로 옮겨진다.
4. **세부 단계 관리** — 각 스테이지에는 세부 단계 체크리스트가 있다
   (예: 상담 = 니즈 파악 콜·기술 미팅·현장 방문 / 협상 = 가격 협상·법무 검토·이사회 승인).
   항목을 클릭해 완료 토글하면 카드 진행률 바와 "다음 액션"이 갱신된다.
5. **메모** — 모달의 메모란은 입력 즉시 저장되며 다시 열어도 유지된다(세션 로컬).
6. **견적 재발행** — 모달 하단 버튼 → 견적 모달이 딜의 랙 수로 프리필된 채 열린다.
7. **계약 전환** — **"계약 전환" 스테이지에서만** 활성화된다(이전 단계에서 누르면 경고).
   확정 시 Control-Plane에 **딜의 실제 랙 수로 테넌트 + 개통 주문(승인 모드)**이 생성되고,
   "운영 콘솔 승인 후 개통" 안내가 표시된다. 계약 관리 화면에 "개통 중" 행이 등장한다.
8. **실주 처리** — 사유 입력(필수) 후 확정하면 보드에서 제외된다.

### 대표 시나리오 — 리드에서 랙 할당까지 (검증 완료 경로)
```
+리드 등록 (acme, 4랙)
  → 카드 클릭 → 상담 (니즈 파악·기술 미팅 체크)
  → 제안/견적 (견적 재발행 → 견적 모달 발송)
  → 협상 (가격 협상·법무 검토 체크)
  → 계약 전환 스테이지 → [계약 전환] 확정
  → Control-Plane: tnt-acme + 주문(4랙, 승인 대기) 생성
  → 운영 콘솔에서 승인 게이트 7단계 통과
     (validated→reserved→provisioning→isolating→storage_binding→acceptance→delivered)
  → NICo가 랙×18 트레이에 DPU 테넌트 격리·세그먼트(VRF/L3VNI) 적용
  → 계약 관리에 활성 계약 + 매출 화면에 과금 라인 자동 생성
```

### 라이브 / mock 구분
- **라이브**: delta-corp 딜 상태·전환(`convertDeal`), 보드 딜의 계약 전환(Control-Plane 직접 주문),
  계약·과금 반영.
- **세션 로컬**: 보드의 스테이지 이동·세부 단계 체크·메모·신규 리드 (새로고침 시 시드로 복원).

## 2. 견적 · RFQ (`#/rfq`)

### 화면 구성
| 영역 | 내용 |
|------|------|
| RFQ 접수함 | RFQ 목록(마감일·랙 수·예상 TCV 칩) + 상태 필터(전체/대기/완료) |
| 견적 자동산정 | 랙 수·계약 기간 입력 → 월액/TCV 자동 계산 |

### 사용법
1. **RFQ 접수** — `+ RFQ 추가`로 신규 RFQ를 접수한다(세션 로컬).
2. **견적 산정** — RFQ 항목의 `견적 산정`을 누르면 산정기에 랙·기간이 로드된다.
   단가는 Control-Plane **billing/rates 라이브 값**($980/rack-h, 캡션에 "라이브" 표시)을 쓰고,
   미연동 시 정적 단가표 v2.4로 폴백한다.
   계산식: `랙 수 × 단가 × 기간 할인 × 720h → 월액 · 36개월 TCV`.
3. **응답(견적 발송)** — 견적 모달이 프리필된 채 열린다. 발송 후 `응답 완료 처리`로 상태를 닫는다.
4. `견적서 PDF`·`CRM에 저장`은 데모 동작(토스트 라벨 표기).

## 3. 계약 관리 (`#/contracts`)

### 화면 구성
| 영역 | 내용 |
|------|------|
| 계약 목록 | 고객/규모/MRR/잔여 TCV/기간/상태 6열 — **라이브 `contracts()`** (테넌트+주문+단가 합성) |
| 갱신·해지 알림 | 갱신 임박(D-일) 계약 하이라이트 |
| 표준 조항 | SLA·크레딧 조항 요약 |

### 사용법
1. **개통 추적** — 파이프라인에서 전환된 딜은 자동으로 **"개통 중 — ord-X 승인 게이트"** 행으로
   나타나고, 운영 콘솔 승인이 진행되면 상태가 갱신되어 활성 계약으로 전환된다
   (`provision.approved` 이벤트 실시간 수신).
2. **MRR 기간 환산** — 상단 기간 토글(월간/분기/연간)에 따라 MRR 컬럼이
   MRR/분기액/연액(ARR)으로 환산 표시된다.
3. `+ 계약 등록`(수기 등록)·`플레이북 →`은 데모 동작.
